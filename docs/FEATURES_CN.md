# 功能说明

简体中文 | [English](FEATURES.md)

Codex QQ Bot 是本地运行的消息中枢：OneBot 负责 QQ 协议，Hub 负责权限、上下文、工具、记忆与投递，Codex CLI 负责理解和执行任务。仓库不分发 QQ、NapCat 或 LLBot 二进制。

## QQ 接入与触发

- 支持 OneBot 11 HTTP API 与反向 HTTP Webhook。
- 群聊只处理 `qq.allowedGroups` 白名单；私聊和群聊使用独立上下文。
- 普通群消息默认只在 @Bot、回复 Bot 或拍一拍 Bot 时触发。
- 白名单群中的已识别斜杠指令不要求 @；主动兴趣回复使用独立、受限的触发链。
- 消息 ID、QQ 号和群号会归一化；重复 OneBot 事件会在进入业务逻辑前丢弃。
- 当前 scope 的回复上下文会用“当前群名片/昵称 + QQ号”标识真人发送者。`@` 目标会保留 QQ 号；消息段没有昵称时，再从当前群成员资料补全。同一 QQ 号在不同群中是同一个稳定人物身份，但各群群名片仍分别保存和使用。
- JSON/XML 分享卡片和合并转发会转成有界、可读的非可信上下文。

## Codex Agent 回复链

所有正常 QQ 回复走同一 Agent 路径：

```text
触发判断
  -> 最近聊天、相关历史、图片与记忆
  -> Codex 首轮判断
  -> 可选多轮内部工具
  -> 最终可见回复
  -> 文本/图片/文件/贴纸/多气泡投递
```

简单对话可以一轮完成；需要更多聊天记录、联网事实、记忆或管理动作时进入有上限的内部工具循环。工具继承原始发送者权限，不能借助模型获得主人权限。`[[qq_done]]`、`[[qq_command:...]]`、记忆 patch 等隐藏 marker 会在发送前校验并移除。

每个由模型处理的回复轮——包括私聊、明确 @、回复 Bot、拍一拍以及已批准的主动轮——都保留最终不回复的选择。互动没有实质内容、重复到令人厌烦、不再有价值，或 Bot 确实不想继续时，模型只输出 `[[qq_silent]]`；Hub 把本轮记录为 `silent` 并完全不投递。确定性的管理斜杠指令仍返回执行结果。任何可见文字如果声称已经完成真实 QQ 写操作——包括拍一拍、加好友、加群、处理申请、群管理或 QQ 空间写入——都必须先调用对应工具并拿到明确成功结果。Bot 可以在调用工具前拒绝，但已经成功的动作必须反馈，失败时绝不能假装成功。反拍必须先调用 `[[qq_command:/拍一拍 发送者]]`；若模型偶发漏写工具标记，Hub 也会识别“反拍一下”“拍回来”“反手戳”等常见说法，在保留可见动作描述前补做对应 OneBot 动作。

默认最多同时运行 2 个 Codex 子进程，并保留 32 个等待任务。每个群或私聊 scope 同时只有一个完整回复生命周期。回复生成期间，所有真正触发 Bot 的后续消息——直接 @、回复 Bot、兴趣模型批准或其他响应规则——进入同一个融合缓冲；900ms 静默窗口内持续到达的消息会重置等待，但单批最多等待 2.5 秒。Hub 把触发消息、它们之间由原上下文规则筛出的附近语境、连续复读和图片合成一份输入，只调用一次 Codex `turn/steer`。同一轮不为每条追问重复携带完整上下文；若追问刚好错过当前 Codex 轮次，发送前会以尚未投递的草稿再融合修订，尽量仍只发送一份回复。`/stop` 只中断当前回复并清理本轮待融合队列，保留滚动上下文、短期记忆和可续用 Codex 线程；`/新对话` 才会额外清除这些会话层并切断线程映射。

QQ 回复支持 `临时`、`长期` 和 `自动` Codex 会话模式。临时模式每个顶层回复使用独立线程；长期模式用 `thread/resume` 续用当前群或私聊线程，只补尚未见过的增量语境并排除当前触发批次的重复内容；自动模式在 6 小时内已有 3 次回复、24 小时内已有 5 次回复，或存在 72 小时内使用过的线程时切到长期，否则使用临时。三种模式共用完全相同的融合追问逻辑。主人可用 `/会话模式 自动|长期|临时` 按当前群/私聊设置；`/新对话` 会切断线程复用。

群回复正文中的 `@准确昵称 ` 或 `@QQ号 ` 会转换为 OneBot 的真实 `at` 消息段。Hub 会从当前/近期身份和有缓存的群成员列表中解析准确昵称；重名时要求使用 QQ 号，不会冒险艾特错人；每个气泡最多解析 8 个显式目标，每个真实艾特后固定补一个普通空格。模型显式指定了目标时，该气泡不再重复添加系统自动选择的发送者艾特。

Codex 时限按任务类型选择，而不是所有工作共用一个常量。默认普通回复 2 分钟、看图回复 3 分钟、上下文总结 90 秒、自我人格任务 90 秒、主人文件任务 5 分钟、图片生成 10 分钟；每一类都能通过独立环境变量调整。运行状态和日志会记录实际任务类型与采用的时限，`/stop` 仍可随时提前终止。

## 上下文与记忆

项目把不同寿命和隐私边界的记忆分开：

| 层级 | 内容 | 存储 |
| --- | --- | --- |
| 滚动上下文 | 当前群/私聊的最近人类消息与 Bot 回复 | `data/qq-memory.json` |
| 会话记录 | 从最近一次 `/新对话` 开始的有界对话和图片引用 | 运行状态与本地持久文件 |
| Codex 会话映射 | 自动/长期 scope 的线程、模型与已注入语境位置 | `data/qq-codex-sessions.json` |
| 短期记忆 | `/记忆` 管理的当前群/私聊临时事项；随 `/新对话` 清除 | `data/qq-memory.json` |
| 社交记忆 | 按群隔离的群印象，以及按 QQ 号共享的有界跨群人物印象 | `data/qq-conversation-memory.json` |
| 长期知识库 | 带标题的稳定知识，以及按全局/群/人物/群内人物分类的黑话 | `data/qq-knowledge-base.json` |
| 旧公共记忆 | 启动时一次性导入长期知识库的兼容数据 | `data/qq-public-memory.json` |
| 自适应统计 | 群节奏、结构风格、成员交互统计 | `data/qq-personas.json` |
| 全局自我人格 | 各 scope 匿名摘要生成的安全人格 | `data/qq-self-persona.json` |
| 统一记忆 | QQ 与可持久化的本地上下文 | `src/unified-memory/` 对应数据文件 |

`/stop` 保留滚动上下文、当前 scope 的 `/记忆` 短期项和可续用 Codex 线程；若长期线程刚启动，也会先保留映射再中断。`/新对话` 清除滚动上下文、会话记录、当前 scope 的 `/记忆` 短期项和 Codex 线程映射，但保留社交印象和长期知识库。它只停止复用旧 Codex 线程，不主动删除 Codex CLI 在本机保存的历史归档。群印象只属于当前群；同一 QQ 号的稳定、非敏感人物印象可以跨共同群复用，但不会复制原始群聊或群内私密事项。旧版按群保存的人物印象自动迁移到 v2 身份层，旧公共记忆自动导入知识库。

原始聊天记录仍逐条保存，供频率统计、时间判断和审计使用；只有组装模型输入时才压缩连续复读。相邻且语义结构相同的消息从 2 条起合并，只保留最后一条作为当前发送者代表，并在正文末尾标注“（连续重复 N 条）”。多个复读段分别压缩，不连续的同文消息不合并；Bot/真人角色、@ 目标、引用或图片不同也不会误合并。该规则统一用于主回复上下文、兴趣判断、冷群/私聊上下文、聊天与人格总结、聊天记录工具和知识删除证据。

知识库每项必须有标题。黑话包含词和解释：同词可分别保存全局、群、人物和群内人物含义，群范围保留群名/群号，人物范围保留昵称/QQ 号；同一人物在多个群出现相同解释时会合并成跨群人物含义。当前消息命中黑话后，Hub 只把当前群和当前发送者适用的解释交给主模型与兴趣模型。`/知识库 标题`、`搜索`、`查看`、`添加` 和 `黑话` 提供隐藏内部工具入口，可指定当前、群、人物或主人专用的跨范围查询。

普通知识不是固定栏目。每轮范围总结会同时读取上轮摘要与主要话题，把它们当作更早聊天的压缩证据，再结合新聊天保留、调整或移除主题；日常主模型也会收到当前范围摘要作为弱参考。因此知识选题来自这个群长期实际主要聊的内容，并会随群聊重心演变，不会预设或永久套用任何领域。只围绕这些真实主话题保存以后会复用的事实、资料、经验或约定，不同群会自然形成不同知识内容。外部且会变化的知识使用不含日期/版本号的稳定标题，正文带截至日期、核验状态、事实和来源，条目展示最近更新时间。带工具的主模型在相关话题再次出现、旧内容可能过时或用户要求最新资料时，先查旧标题并使用已有联网摘要或主动搜索核验，再用同一标题和范围覆盖更新；来源冲突时不写成定论。无联网工具的总结任务只能把聊天说法标成“会话待核查”，不能伪装成已联网验证。群内规则等不可由公网核验的内容会明确标成群内约定或共识。

总结和印象更新会拿到当前范围已有知识：相同标题/范围直接覆盖旧内容，同义改名保留别名，避免只追加不更新。黑话命中会累计次数、首末时间，并有界保留命中消息及前后三条周边聊天。较旧且长期低频的黑话只会形成删除申请；兴趣模型先读取统计与少量首尾样本，做低温初筛和风险标注，随后主模型读取全部保留证据作最终保留/删除判断。任一模型失败都不删除；审核期间一旦出现新命中或内容更新，Hub 也会自动保留，防止过时审核误删。知识文件加载异常时保留原文件并阻止写入。

所有周期性 QQ 行为都按持久化的墙上时间判断，不依赖进程连续运行时长。Hub 启动时会立即检查一次，QQ 通道重新启用时再立即检查一次，之后才按配置的轮询间隔检查。设备或 Hub 停止期间如果越过到期时间，恢复后只补做一轮，不会把错过的每个周期集中连续回放；补做完成后以完成时刻为新起点重新计时。普通群兴趣会把 pending 数量、周期起点和有界的最后候选消息写进 `data/qq-memory.json`；风格复盘、冷群/私聊兴趣、自我人格摘要与生成继续使用各自 persona 文件中的持久时间戳。手动 `/总结聊天记录` 和统一记忆读写仍是事件触发，不会被人为改成没有意义的周期任务。

## 自适应社交行为

- `qq-human-behavior` 从近期人类消息统计长度、速度、连续发言、图片/贴纸/emoji、回复/@、问句和标点，不复制某个成员原话；主模型只接收本轮动态行为规划，不再叠加一套固定“群聊风格”。
- `qq-adaptive-learning` 持久记录活跃时间、消息结构、交互距离和 Bot 后续反馈，弱调整回复长度、贴纸概率、多气泡节奏与延迟。它还统计两条真人消息相隔不超过 2 分钟时的活跃衔接，并以其中发送者发生切换的比例作为群级“插话率”。
- 每个群有 24 小时风格复查时钟；样本足够时生成最多五条压缩改进建议，并替换旧建议避免无限增长。
- Bot 的全局自我人格只使用匿名 scope 摘要生成，禁止把一个私聊中的原始私人内容复制到其他群或私聊。
- 自适应统计只影响“怎么回复”和触发节奏，不能绕过群白名单、权限或兴趣 judge。

## 主动兴趣

主动回复分为三个互相受限的分支：

1. **普通群聊兴趣：**未 @ 的普通消息进入持久化的每群 pending cycle；消息数或非空周期的墙上时间分钟阈值先满足者触发已配置厂商的语义 judge。空周期不调用模型；重启后会立即检查已到期的非空周期。从磁盘恢复的周期仅在这次补做 judge 中绕过正常在线的“旧话题丢弃”保护，避免长时间停机后把到期行为静默消费掉；judge 期间出现的新消息仍可使旧结果失效。
2. **冷群兴趣：**根据群活跃度、最近人类/Bot 消息、未回应次数和学习到的活跃时间窗进入候选状态后，由兴趣模型返回 `silent`、`topic` 或 `chatter`。拒绝或判定失败时不会启动主模型；批准 `topic` 后主模型依据自己的全局兴趣、长期记忆和最近群话题自由选题，可使用现有 Agent 工具进行多轮联网搜索或资料探索；`chatter` 则执行少见的轻量水群。兴趣模型不提供具体题目、搜索词、回复草稿或聊天风格。
3. **私聊兴趣：**交互频率、距离上次活动时间和未回应 Bot 消息先形成频率先验与拟人波动值，再由兴趣模型作最终启动判定。拒绝或模型失败时不启动主模型；批准后主模型只写自然私聊内容，不再重新判断是否联系。

普通群聊 judge 使用流式兼容请求：OpenRouter 使用严格 JSON Schema，DeepSeek 和自定义 OpenAI 兼容厂商使用 JSON Object 模式；最终都只接受 `shouldReply`、`interest` 与 `reason`。它不再输出分析过程、语义总结、回复草稿或风格建议。原消息、引用和最近上下文直接交给获准后的主模型理解，兴趣模型的理由不能覆盖原始语境。普通群判断温度为 `0.65`，冷群和私聊启动判断为 `0.8`，让兴趣出现适度波动；复杂删除审核的兴趣初筛使用 `0.15`。实际温度和模型原始结构化输出写入详细日志。结构无效时最多重试一次；超时、HTTP 错误和限流不会盲目重试。

兴趣模型是后台轻量判断与杂项初筛层：普通主动回复闸门、冷群/私聊启动、短小分类、风险标注和复杂任务的第一轮有界初筛走同一个已配置通道。它不生成面向群友的聊天内容，也不承担聊天总结、群/人物印象总结、人格总结或知识内容提取。主模型是聊天、总结和复杂任务层，负责正式回复、语境理解、工具检索、选题表达、所有总结/知识提取，以及长上下文或多证据任务的最终处理。

兴趣 judge 会收到自动适应学习出的群级插话率、活跃衔接样本数和统计窗口。它们只用于判断接话时机：高插话率不会直接触发回复，低插话率也不会形成硬性禁言；最终仍由模型结构化结论和兴趣阈值决定。

主动分支不会排在正在生成的任务后强行插话；新活动会抑制过时结果。普通主动接话、冷群自主话题、冷群轻量水群和主动私聊都必须携带统一的“兴趣模型批准 → 主模型内容”契约；缺少任一阶段时 Hub 会阻止发送。主模型提示词按“角色—理解顺序—作答—记忆—安全—动态风格—当前任务”组织，主动审批只出现一次；常用查询工具与偶发社交工具按当前消息相关性展示，避免无关工具说明挤占聊天注意力。双模型契约、冷群/私聊启动判定、模型温度、具体输出、冷群探索轮数、工具类型、搜索词、失败数和最终结果都记录在详细日志。

## 图片、文件、贴纸和多气泡

- 识别 QQ 图片、截图、引用图片、分享卡片和有界合并转发图片。
- 明确的画图/生图请求，以及“编辑、修改、换背景、增删元素、改风格、按这张图再画”等参考图编辑说法，会优先进入图片任务而不是被当成普通看图。当前消息或引用消息有图片时，即使只说“把背景换成蓝色”，图片也会作为参考输入交给画图模型；纯“看看这张图”仍走视觉理解回复。
- 回复模型可使用 `[[qq_image:...]]`、`[[qq_file:...]]` 和 `[[qq_sticker:...]]` 请求本地投递。表情包支持三种逐回复选择：与文字合并为同一条 QQ 消息、只发表情包，或在文字与表情包之间使用独立一行的多气泡分隔符后按两条消息顺序发送。
- 文件/生成图任务使用 `runtime/qq-task-workspaces/<request>/input|output`；只有当前请求 `output/` 内通过真实路径校验的文件可投递，防止任意本地文件泄露和符号链接逃逸。
- 贴纸目录、QQ 账号收藏和下载元数据组成有界 catalog；动画可按百分比或抽帧检查，持久标签单独保存。
- 在已有回复生命周期中，模型可以决定是否收藏当前贴纸；普通未触发贴纸不会单独调用模型。
- 独立一行 `|||` 可分隔连续气泡；用于表情包时，文字气泡先发送、仅含表情 marker 的气泡随后独立发送。数量、分隔符和延迟可配置，自适应层只在适合的社交回复中调整节奏。

## QQ 指令、权限与社交工具

常用公开指令包括：

- `/菜单`、`/新对话`、`/stop`、`/总结聊天记录`。

主人还可用 `/会话模式` 查看当前 scope 的配置、实际模式和自动判断计数，并用 `/会话模式 自动|长期|临时` 修改。

默认主人管理能力包括：

- 状态与详细配置；模型和思考强度。
- 主动兴趣参数、群白名单、ban 和指令权限。
- 群禁言、解禁、踢人和全员禁言。
- 好友/入群请求同步、同意、拒绝与通知。
- 可用时通过本机 NapCat 社交扩展发起好友/加群请求，以及读取或写入 QQ 空间动作。主动加好友工具除规范的 `/主动加好友` 外，也识别 `/加好友` 和 `/添加好友`。好友桥接把陌生人的 UID 查询作为可选预检，查询失败仍以 QQ 号调用原生接口；`/inspect-friend` 提供不发送申请的本机只读诊断。NapCat 4.18.9 的主动加好友调用双参数签名 `reqToAddFriends(QQ号, 验证文本)`，并按验证模式选择验证信息或问题答案。返回 `submitted` 不代表对方已经同意。
- QQ 空间评论使用当前 `topicId` 和字符集字段。主人可用 `/发动态 文字`、在当前/引用消息带图时用 `/发动态 图片=当前`，或 `/发动态 文字 | 图片=当前` 发布最多 9 张图片；Bot 只读取本轮安全临时目录中的图片，不接受任意文件路径。

非主人能看到的菜单项与实际可执行权限来自同一组 permission key。主人 QQ 号和 Bot 自身不能被非主人降权、ban、禁言或踢出。

## 联网搜索

Hub 自带 QQ 联网查询，不依赖当前聊天界面的浏览工具。支持按配置顺序尝试 Tavily、Bing、百度、360、搜狗和 DuckDuckGo；预设包括 `balanced`、`china`、`global`、`tavily` 和 `privacy`。查询结果作为非可信资料进入 Agent，不能覆盖原始用户权限或系统规则。

实际 provider、尝试记录和最近错误通过 `/api/maintenance` 与 `search` 日志查看。

## 仪表盘、API 与日志

本地仪表盘提供：

- Hub、Codex、OneBot 与 QQ 通道状态。
- 群白名单、模型、短期记忆、自适应学习和主动兴趣状态。
- 独立的长期知识工作区：按黑话/知识和全局/群/成员/群内成员筛选，查看同词的不同解释、命中频率、最近语境与模型审查结果，并精确增删改当前范围解释。
- 维护信息、结构化日志筛选、主题、语言和响应式布局。
- 持久局域网开关；开启时自动创建管理 token，token 只允许从本机页面读取。
- 默认关闭的 Cloudflare Quick Tunnel 开关；不改变 Hub 的回环监听即可生成临时 HTTPS 地址，远端管理 API 仍需同一 token，隧道启停和 token 复制只允许本机页面操作。

自动轮询只更新服务端读数，不会覆盖正在操作的控件或尚未保存的 Bot 设置。当前浏览器标签页会用会话存储保留 Bot 设置草稿、群 ID 输入、短期记忆页签/搜索/展开项、知识筛选/选择，以及日志筛选、暂停、跟随和滚动位置；整页刷新后会恢复这些编辑上下文。知识编辑使用条目 ID 与范围解释 ID 做过时保护，删除只作用于当前范围；保存成功后以服务端返回状态为准。

核心只读入口：

```text
GET /api/state
GET /api/maintenance
GET /api/logs
GET /api/memory
POST /api/qq/knowledge
```

日志采用 JSONL，支持 level、category、trace、group、sender、关键词、时间和延迟过滤。单条 QQ 生命周期共享 trace，便于从入站、judge、融合追问、搜索、Codex 到投递定位问题。融合日志记录缓冲、一次性 `turn/steer`、发送前兜底、触发来源、原始/压缩数量、中间语境数、图片数和有界内容预览；`ncc logs` 沿用现有彩色级别/分类字幕和中文字段。短期记忆变更、知识库查询/更新、黑话命中频率、低频删除申请与模型审核都使用同一 `memory` 分类和结构化详情。Codex 与兴趣模型的具体输出会以 `debug` 详情进行长度限制和密钥脱敏后记录，但不会复制完整输入提示词；中文终端和网页日志会递归中文化启动时的自动学习快照及其他详情字段。

## macOS 客户端

原生 macOS 客户端只是同一本地仪表盘与 QQ/OneBot Hub 的 WebKit 外壳，和浏览器使用完全相同的链路。它不读取 Messages 数据库、不轮询 iMessage、不要求“信息”自动化或完全磁盘访问权限，也不再提供 macOS 独有的代理、显示器、防休眠或桌面控制能力。

## 明确边界

- 不是官方 QQ Bot OpenAPI 服务，也不是托管式公网 Bot。
- 不包含 QQ、NapCat 或 LLBot 安装包。
- 不会自动绕过 QQ 风控、验证码、扫码或系统权限。
- 默认不开放公网/局域网，不允许无认证远程管理。
- 自适应行为不会模仿或公开某个成员的原话与私人事实。
